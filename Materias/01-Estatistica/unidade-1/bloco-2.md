## Estatística descritiva
A estatística descritiva tem como objetivo organizar e analisar um conjunto de dados utilizando:
- Média 
- Moda
- Mediana

Você não tenta prever nada, nem generalizar.

Só trabalha com os dados

Ferramentas principais:

### Média 
É a soma dos valores dividida pela quantidade.

Exemplo:
- Notas 6, 8, 10
- Média = (6 + 8 + 10) / 3 = 8

### Moda 
É o valor que mais se repete

Exemplo:
- 2, 3, 3, 5, 7
- Moda = 3

### Mediana
É o valor que fica no meio, depois de ordenar os dados

Exemplo: 
- 1, 3, 5, 7, 9
- Mediana = 5

Se tiver um número par de dados, faz a média dos dois do meio.

Deve se basear no método estatístico, que é composto por uma série de etapas listadas a seguir:

- Definição do problema
- Delimitaçã odo problema
- Planejamento para a obtenção dos dados
- Coleta dos dados
- Apuração dos dados
- Apresentçaão dos dados
- Análise dos dados
- Interpretação dos dados.

## Estatística indutiva
Você usa dados para tirar conclusões maiores.

Você pega uma amostra e tenta entender a população inteira.

Nessa, vamos supor que você não tem todos os dados, então você estima com base no que tem

Exemplo:
- 70% da amostra prefere algo
- Você conclui que provavelmente ~70% da população também prefere

A ideia central é:
- fazer previsões
- generalizar resultados
- trabalhar com probabilidade

Diferença das duas:

Estatística Descritiva:
- Trabalha com os dados coletados
- Não faz previsão
- Usa média, moda, mediana

Estatística Indutiva:
- Usa amostra
- Faz estimativas sobre a população
- Envolve probabilidade

## Dados brutos
São os dados exatamente como foram coletados, sem nenhuma organização ou tratamento.

São os dados crus.

Exemplo simples:
Imagine que você coletou salários de fucionários:

3000, 1500, 4200, 1500, 2800

Isso é dado bruto porque:
- Não está organizado 
- Não foi analisado
- Não passou por nenhum tipo de tratamento

Características:
- podem estar bagunçados 
- podem ter erros
- podem ter inconsistências

Eles podem estar inconsistentes ou padronizados de formas diferentes, o que atrapalha a análise

Por exemplo:

1- Datas em formatos diferentes
- 10/03/2025
- 03-10-2024
- 2024/03/10

Todos apresentam a mesma coisa, mas estão diferentes.

2- Nomes escritos de formas diferentes
- São Paulo
- S. Paulo
- SP

Todos significam o msm lugar, mas o sistema pode interpretar como diferentes.

Então antes de analisar, você precisa tratar e padronizar os dados

Isso inclui:
- corrigir erros
- padronizar formatos
- organizar informações

Só depois disso os dados ficam prontos para análise.

## Rol
O Rol é simplesmente - dados organizados

Você peg aos dados brutos (bagunçados) e coloca em alguma ordem.

Ideia principal:
- Dados brutos -> sem organização
- Rol -> dados organizados

### Formas de organizar
Os dados podem ser organizados em três formas principais:
- Crescente (do menor pro maior)
- Decrescente (do maior para o menor)
- Ordem alfabética (quando não são números)

Exemplos: 

Dados brutos (bagunçados):  
9, 4, 6, 7, 8,5, 3, 8, 10, 10

Agora organizando:

Ordem crescente:  
3, 4, 6, 7, 8, 8,5, 9, 10, 10

Ordem decrescente:  
10, 10, 9, 8,5, 8, 7, 6, 4, 3

A partir do momento que organizou → virou Rol.

Ele é importante, pois podemos:
- ver valores mínimos e máximos
- identificar repetições (moda)
- encontrar mediana
- entender a distribuição dos dados

O rol ajuda, mas não é o nível mais avançado de organização.

### Dados não númericos
Nem todo dado é número.

Exemplo: nomes dos alunos

Isso é chamado de: dados alfanuméricos

E também pode virar Rol

Exemplos:

Nomes:
Maria, Pedro, Otávio, Mariana

Rol:
Leonardo, Maria, Mariana, Matheus, Otávio, Pedro...

Se é dados organizados em ordem crescente, decrescente ou alfanumérico - é rol.

Se for "dados sem organização" são dados brutos.
